Git的工作区域和文件的状态

工作区域

  • 工作区域

就是我们写代码的区域，和正常的目录没有什么区别

  • 暂存区域

保存着临时修改的代码，这时候的代码已经被git进行了管理

  • 本地仓库

最终版本存储的代码

文件状态

需要注意未修改、已修改、已暂存是被git控制的

添加和提交文件

添加

git add 之后的文件会变成绿色

除此之外还可以使用通配符来进行添加多个文件

还可以使用

提交

git commit 命令只能提交暂存区的文件

提交完成之后暂存区就没有文件了

可以使用git log命令来看历史的命令

git reset 回退版本

对号：保留内容

错：删除区域内的内容

直接ls就可以查看工作区的内容

hard场景最好不要使用，如果不小心操作失误了可以使用git reflog 查看历史的记录，记录下版本号然后在git reset –模式 版本号 ，就可以了

git diff 查看版本之间的版本差异

如果不加参数默认比较工作区和暂存区域

这里默认比较的是工作区和暂存区的文件，如果我们把修改之后的文件提交到暂存区在使用git diff就不会有任何的显示

如果想要比较工作区域+暂存区和本地仓库，使用

如果想要比较暂存区域和本地仓库使用

还可以比较不同版本之间的差异

每次查看版本号太麻烦可以使用HEAD~1/2/3

git diff 还可以查看版本之间某一文件的差异

在版本库中删除文件

现在工作区域删除，然后在暂存区域删除

上面有点麻烦，可以直接使用girt rm 命令，这样会同时删除暂存区和工作区域，如果我们不提交删除的文件在版本库中还是存在的！！

克隆成功

我们一顿操作下来可以发现并没有添加到远程的仓库

我们需要使用git push 把本地修改的内容推送到远程仓库

关联本地仓库和远程仓库

我们自己创建了一个本地仓库learn_git，同时远程新创建一个仓库

使用git remote add 命令把本地的仓库和刚刚在github创建的仓库关联起来

git remote -v 可以查看当前仓库对应的远程仓库的别名和地址

我们在远程仓库修改之后可以只用git pull 拉去远程改变之后的仓库

仓库的名称和分支的名称可以忽略
